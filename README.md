# 🏎️ 创客社科技节 - 双队并行计时系统

这是一个专为创客社科技节（小车组装比赛）设计的实时计时与排位管理系统。系统采用 B/S 架构，使用 `uv` 进行极速的环境与依赖管理，支持**双队伍并行计时**、**分段独立耗时记录**、实时大屏展示以及动态排位计算。

## ✨ 核心功能

- **👥 双队并行控制**：在同一界面同时监控和控制两个队伍的比赛进程，互不干扰。
- **⏱️ 精准分段计时**：
  - 支持“组装”、“运行”、“拆解”三个步骤的独立计时。
  - **核心逻辑**：步骤耗时 = 本步骤结束时间 - 上一步骤结束时间（而非累计总时间）。
- **🏆 实时排位榜单**：
  - 自动计算各步骤的单项排名。
  - 自动计算比赛总用时排名。
- **🖥️ 可视化大屏**：现代化的深色渐变 UI 设计，适配投影仪与大屏展示，状态实时动态更新。
- **🔌 RESTful API**：提供完整的后端接口，支持队伍增删、计时控制、状态查询等操作。

## 🛠️ 技术栈

- **环境与依赖管理**：uv (极速 Python 包管理器)
- **后端**：Python 3.x, Flask, Flask-CORS
- **前端**：HTML5, CSS3 (Flexbox/Grid, Animations), JavaScript (ES6+, Fetch API)
- **数据存储**：内存存储 (基于 Python 类对象，线程安全)

## 🚀 快速开始

### 1. 环境准备
确保你的计算机已安装 `uv`。如果尚未安装，可以通过以下命令快速安装：
```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 2. 初始化并安装依赖
在项目根目录下，使用 `uv` 初始化项目并安装所需依赖（Flask 和 Flask-CORS）：

```bash
# 添加项目依赖
uv add flask flask-cors

# 创建并激活虚拟环境
uv venv
# Windows 激活命令
.venv\Scripts\activate
# macOS / Linux 激活命令
source .venv/bin/activate
```

### 3. 运行后端服务
在虚拟环境激活的状态下，执行以下命令启动 Flask 服务器：
```bash
uv run python timer_backend.py
```
服务默认运行在 `http://127.0.0.1:5000`。

### 4. 访问大屏
打开浏览器，访问 `http://127.0.0.1:5000` 即可进入双队计时大屏界面。
*首次访问时，系统会自动创建“队伍 A”和“队伍 B”两个初始队伍。*

## 📡 API 接口说明

后端提供标准的 RESTful API，主要接口如下：

| 请求方式 | 接口路径 | 说明 |
| :--- | :--- | :--- |
| `POST` | `/api/teams` | 添加新队伍 (Body: `{"name": "队伍名"}`) |
| `GET` | `/api/teams/status` | 获取所有队伍的实时状态（用于前端轮询） |
| `POST` | `/api/teams/<id>/start` | 开始指定队伍的计时 |
| `POST` | `/api/teams/<id>/pause` | 暂停指定队伍的计时 |
| `POST` | `/api/teams/<id>/stop` | 停止指定队伍的计时 |
| `POST` | `/api/teams/<id>/record_step/<step>` | 记录步骤完成 (step: 1组装, 2运行, 3拆解) |
| `GET` | `/api/rankings/total` | 获取总计时排位 |
| `GET` | `/api/rankings/step/<step>` | 获取指定步骤的单项排位 |
| `POST` | `/api/reset` | 重置所有队伍状态 |

## ⚙️ 计时逻辑说明

本系统采用**纯分段计时**逻辑，确保每个步骤的耗时独立计算：
1. **组装耗时** = 记录步骤1时的总耗时 - 0
2. **运行耗时** = 记录步骤2时的总耗时 - 记录步骤1时的总耗时
3. **拆解耗时** = 记录步骤3时的总耗时 - 记录步骤2时的总耗时

当三个步骤全部记录完成后，队伍状态会自动变更为“已结束”，并锁定总用时。
